# `nubot_plugin.cc` 函数说明

## [`NubotGazebo::dribble_ball`](./nubot_plugin.cc#L539)

`dribble_ball` 用于在机器人满足带球条件时模拟控球效果。该函数由
`nubot_be_control` 调用，在每个控制周期内更新足球的位置和运动状态，使足球保持在
机器人前方并随机器人移动。

函数的处理过程如下：

1. 检查 `ball_entity_` 是否有效；如果尚未获取足球实体，则直接返回，避免访问无效实体。
2. 根据世界坐标系中的踢球方向 `kick_vector_world_` 计算机器人前方 `0.35 m` 处的相对位置。
3. 将相对位置与机器人当前位置相加，得到足球的目标位置；当 `flip_cord_` 为真时，
   对目标坐标取反，以适配场地两侧使用的翻转坐标系。
4. 将足球目标高度固定为 `0.12 m`，并使用机器人的朝向构造足球目标位姿。
5. 通过 Gazebo ECS 的 `Pose` 组件把足球移动到目标位姿，同时将足球的
   `LinearVelocity` 组件清零，防止原有速度使足球脱离控球位置。
6. 将缓存的足球线速度 `ball_state_.twist.linear` 更新为机器人当前线速度，保持插件内部
   状态与机器人带球运动状态一致。

参数 `_ecm` 是 Gazebo 仿真的实体组件管理器，用于创建或更新足球实体的位姿和线速度组件。
